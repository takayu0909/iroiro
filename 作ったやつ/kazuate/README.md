st.session_stateの使い方が大事。

streamlitは何か実行するたびに上から下まで実行されなおすから次に持ち越したいものがある場合はst.session_stateを使わなければならないと学んだ。

しかし実際に使ってみると宣言するために初回だけ宣言されるように条件判定が必要だったり、その条件判定の仕方もst.session_stateというでかい箱の中に～.変数名の変数名が存在するかを見る必要がありAIに質問して理解した。

mistakesの値が１から変化せずそれを解消するために↑のことが必要だった。

正解判定のst.session_state.target==numがうまくいかなかった。理由はint変換しているけれどnumに再代入していなかったことだった。こういうのは気を付けたい。

ここはどのパートをやっているかのコメントアウトをすると自分でもわかりやすいしほかの人が読んだときにわかりやすいかなと思った。

ヒントは正解の数が小さいと範囲が小さくなりすぎたから5以下の時は5を足すようにした。ヒントの幅の決め方が悪かったからこういう風にやる必要があったと思う。
